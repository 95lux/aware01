\subsection{Discussion of Limitations}

\subsubsection{Hardware Revision Defects}
Two pin misassignments were identified in this hardware revision that could not 
be corrected through firmware alone:
\begin{itemize}
    \item \textbf{Gate 03 input (FR-03):} The assigned pin PC9 does not support 
    EXTI interrupts. A polling-based workaround was implemented, but 
    interrupt-driven edge detection cannot be restored without a PCB revision.
    \item \textbf{Slide potentiometer LEDs (FR-04):} PWM-capable timer channels 
    for the fader LEDs were incorrectly assigned during schematic entry, leaving 
    the slider LED indicators non-functional in this revision.
\end{itemize}
Both defects are documented and a corrected hardware revision with the appropriate pin reassignments is already in progress in a dedicated feature branch of the 
project repository.

\subsubsection{Memory Headroom}
DTCMRAM utilisation stands at 98.5\% and main SRAM at 93\% in the current firmware 
build. While sufficient for the implemented feature set, this leaves limited headroom 
for additional DSP algorithms or larger sample buffers. The current recording and 
playback buffer holds \qty{2.5}{\second} of audio at $f_s = \qty{48}{\kilo\hertz}$, 
which is adequate for the tape-style use case but restrictive for longer recordings. 
Extending buffer capacity would require architectural changes such as reducing reverb 
delay line sizes or migrating to external RAM via QSPI.

\subsubsection{Noise Characterisation (EC-05)}
Quantitative noise floor measurements have not been completed for this revision. 
The analogue/digital separation strategy was applied during layout, but its 
effectiveness has not been formally verified through measurement. During in-system 
testing in a Eurorack environment, the noise performance was subjectively sufficient 
and no audible noise floor was observed. A formal measurement using the swept-sine 
method described in Section~\ref{sec:frequency-response} is planned for a future 
revision.
